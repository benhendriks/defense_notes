# Security Policy

## Scope

This repository contains Markdown documentation, command references, and detection patterns — no executable code, no credentials, no live infrastructure.

Security reports in this context cover:

- Malicious or compromised external links in any file
- Accidental inclusion of restricted or proprietary content (exam questions, lab details, NDA-covered material)
- Commands or instructions that could cause unintended harm without adequate warning
- Any sensitive information inadvertently committed (credentials, personal data, internal infrastructure details)

## Reporting

If you find any of the above, open an issue tagged `security` or contact directly via the email on the profile.

Do not open a public issue for anything that could expose sensitive information — use private contact first.

All reports will be reviewed and addressed promptly.

## Out of scope

Theoretical vulnerabilities in tools referenced in this documentation are out of scope — report those to the respective tool maintainers.
